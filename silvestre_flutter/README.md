# silvestre_flutter

Flutter plugin for the [silvestre](https://github.com/enzoftware/silvestre) image
processing library. Every image operation runs in Rust via
[flutter_rust_bridge](https://github.com/fzyzcjy/flutter_rust_bridge), giving native
performance on Android and iOS while keeping a fully idiomatic, async Dart API.

## Features

- **15 image filters**: grayscale, invert, sepia, brightness, contrast, sharpen,
  box blur, Sobel, Gaussian blur, median, Canny edge detection, crop, resize,
  rotate, mirror
- **Image I/O**: load/save PNG, JPEG, BMP from files or memory
- **Histogram analysis**: per-channel and luminance histograms with statistics
- **Async API**: all operations run on a background isolate, keeping the UI
  responsive

## Architecture

```
silvestre_flutter/
├── lib/
│   ├── silvestre_flutter.dart   # Hand-written, idiomatic Dart API (SilvestreImage)
│   └── src/rust/                # Generated flutter_rust_bridge bindings (do not edit)
├── rust/
│   └── src/api/                 # Thin Rust bridge layer over silvestre-core
└── example/                     # Bloc-based demo app (mirrors the web demo)
```

The Rust code under [`rust/src/api`](rust/src/api) is intentionally a **thin binding
layer**, not a second copy of the image-processing logic. It only adapts
Dart-friendly types (JSON parameter strings, byte buffers) and delegates to the
shared [`silvestre-core`](../silvestre-core) crate. `flutter_rust_bridge` requires a
local crate of its own to scan and generate the Dart↔Rust glue, and the plugin needs
a `cdylib`/`staticlib` build that `silvestre-core` (also used by the CLI, FFI, and
WASM crates) deliberately does not carry. Keeping the binding here keeps the core
pure and reusable.

You do **not** edit the generated files in `lib/src/rust/` or `rust/src/frb_generated.rs`
by hand — they are regenerated by `flutter_rust_bridge_codegen` whenever the Rust API
changes.

## Prerequisites

### Rust toolchain

Install Rust via [rustup](https://rustup.rs/):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```

### Android

Install the Android NDK and add the required Rust targets:

```bash
rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android
```

### iOS

Add the iOS Rust target (requires Xcode):

```bash
rustup target add aarch64-apple-ios
```

For the iOS simulator on Apple Silicon:

```bash
rustup target add aarch64-apple-ios-sim
```

## Installation

Add the dependency to your `pubspec.yaml`:

```yaml
dependencies:
  silvestre_flutter:
    path: ../silvestre_flutter  # adjust path as needed
```

## Usage

### Initialize

Call `Silvestre.init()` once before using any silvestre API, typically in `main()`:

```dart
import 'package:silvestre_flutter/silvestre_flutter.dart';

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Silvestre.init();
  runApp(MyApp());
}
```

> `Silvestre.init()` wraps `RustLib.init()` and selects the correct native
> library loader per platform. On iOS/macOS the Rust code is force-linked into
> the plugin framework, so it loads the symbols from the running process rather
> than searching for a standalone framework.

### Load an image

```dart
// From a file path
final image = await SilvestreImage.fromPath('/path/to/photo.png');

// From bytes (e.g. from an asset or network)
final bytes = await rootBundle.load('assets/photo.png');
final image = await SilvestreImage.fromBytes(bytes.buffer.asUint8List());

// From raw pixels
final image = await SilvestreImage.create(
  pixelData,
  width,
  height,
  ColorSpace.rgba,
);
```

### Apply filters

Filters return a **new** `SilvestreImage`; the source is never mutated, so filters
can be chained.

```dart
// Convenience methods
final gray = await image.grayscale();
final bright = await image.brightness(50);
final blurred = await image.gaussian(2.0);
final edges = await image.canny(low: 50, high: 150, sigma: 1.4);
final cropped = await image.crop(10, 10, 200, 200);
final resized = await image.resize(800, 600);
final rotated = await image.rotate(90.0);
final mirrored = await image.mirror('horizontal');

// Generic filter with params
final filtered = await image.applyFilter('median', params: {'size': 3});
```

### Save / encode

```dart
// Save to file
await image.save('/path/to/output.png');
await image.save('/path/to/output.jpg', format: ImageFormat.jpeg);

// Encode to bytes (e.g. for Image.memory or sharing)
final pngBytes = await image.encode();
final jpegBytes = await image.encode(format: ImageFormat.jpeg);
```

### Histogram analysis

```dart
final histogram = await image.computeHistogram();
print('Channels: ${histogram.numChannels}');
print('Pixel count: ${histogram.pixelCount}');

for (var c = 0; c < histogram.numChannels; c++) {
  final stats = histogram.stats[c];
  print('Channel $c: min=${stats.min}, max=${stats.max}, '
      'mean=${stats.mean.toStringAsFixed(2)}, '
      'stdDev=${stats.stdDev.toStringAsFixed(2)}');
}

// Luminance histogram (single channel, BT.601 weights)
final lumHist = await image.computeLuminanceHistogram();
```

## API Reference

### SilvestreImage

| Method | Description |
|--------|-------------|
| `fromPath(path)` | Load from file path |
| `fromBytes(bytes)` | Load from encoded bytes |
| `create(pixels, w, h, colorSpace)` | Create from raw pixels |
| `width`, `height`, `colorSpace` | Image properties |
| `pixels` | Get raw pixel data |
| `applyFilter(name, params)` | Apply named filter |
| `save(path, format)` | Save to file |
| `encode(format)` | Encode to bytes |
| `computeHistogram()` | Per-channel histogram |
| `computeLuminanceHistogram()` | Luminance histogram |

### Filters

| Filter | Convenience method | Parameters |
|--------|--------------------|------------|
| `grayscale` | `grayscale()` | *(none)* |
| `invert` | `invert()` | *(none)* |
| `sepia` | `sepia()` | *(none)* |
| `sharpen` | `sharpen()` | *(none)* |
| `box_blur` | `boxBlur()` | *(none)* |
| `sobel` | `sobel()` | *(none)* |
| `brightness` | `brightness(delta)` | `delta`: int (-255 to 255) |
| `contrast` | `contrast(factor)` | `factor`: double (0.0–…) |
| `gaussian` | `gaussian(sigma)` | `sigma`: double |
| `median` | `median(size)` | `size`: int (odd kernel size) |
| `canny` | `canny(low, high, sigma)` | `low`, `high`, `sigma`: double |
| `crop` | `crop(x, y, w, h)` | `x`, `y`, `w`, `h`: int |
| `resize` | `resize(w, h)` | `w`, `h`: int |
| `rotate` | `rotate(angle)` | `angle`: double (degrees) |
| `mirror` | `mirror(mode)` | `mode`: `"horizontal"`, `"vertical"`, or `"both"` |

> The filter `name` passed to `applyFilter` uses snake_case (e.g. `box_blur`), while
> the convenience methods use camelCase.

## Example app

The [`example/`](example) app is a complete image editor that mirrors the
[web (WASM) demo](../silvestre-wasm): pick an image, stack filters, view a live
histogram, reset, and export the result via the native share sheet. It uses
[`flutter_bloc`](https://pub.dev/packages/flutter_bloc) for state management:

- `EditorBloc` — owns the image lifecycle (load, apply filter, reset, export)
- `HistogramBloc` — computes histograms on demand

Run it from the `example/` directory:

```bash
cd example
flutter run
```

## Building from source

From the repository root:

```bash
# Check Rust compilation
cargo check -p silvestre_flutter_rust

# Regenerate FRB bindings (after modifying the Rust API)
cd silvestre_flutter
flutter_rust_bridge_codegen generate

# Build for Android
cd example
flutter build apk --debug

# Build for iOS (macOS only)
cd example
flutter build ios --no-codesign
```
